Core simulation logic:
- game loop ticks every 250ms
- elapsed code time updates continuously
- patient state changes over time
- orders expire if not acknowledged or completed
- compressor fatigue appears if compressor role is not switched
- pulse checks longer than threshold incur penalties
- wrong timing of epi or shock affects score
- some complications spawn randomly but plausibly
- staff speak using short speech bubble strings based on state

Upgrade the current code simulation game into a 2.5D clinical environment.

GOAL
Create a visually immersive, pseudo-3D code room where the player leads a code by clicking staff, assigning roles, and managing actions in real time.

IMPORTANT:
Do NOT use true 3D rendering.
Simulate depth using:
- layering (z-index)
- scaling based on vertical position
- angled top-down perspective

Preserve the existing simulation engine (reducer, TICK loop, patient state, orders, chaos system).
The visual layer should reflect and control the existing engine.

-----------------------------------
VISUAL LAYOUT
-----------------------------------
Create a top-down angled (isometric-style) room:

- patient bed in center
- monitor at head of bed
- crash cart / defibrillator to one side
- doorway at edge of room
- staff arranged around bed

Simulate depth by:
- scaling objects smaller when higher on screen
- layering objects so lower screen elements overlap upper ones

-----------------------------------
STAFF REPRESENTATION
-----------------------------------
Each staff member should be a visible entity with:
- circular or card-like avatar
- name label
- role label
- selection highlight
- color-coded by type (nurse, RT, intern, etc.)

They should occupy positions around the bed.

-----------------------------------
MOVEMENT SYSTEM
-----------------------------------
When roles are assigned or changed:

- staff smoothly move to role-specific positions
  - compressor → side of bed
  - airway → head of bed
  - defib → monitor/cart side
  - meds/IV → access side
  - unassigned → periphery

Use smooth interpolation (no teleporting unless necessary).

-----------------------------------
INTERACTION MODEL
-----------------------------------
Click on a staff member to select them.

When selected:
- show a radial menu OR side panel with actions:
  - assign role
  - confirm role
  - give medication
  - ask update
  - redirect

Click on:
- patient → pulse check, rhythm check, airway actions
- defibrillator → charge, shock
- room edge → clear room

No typing allowed.

-----------------------------------
ANIMATION STATES
-----------------------------------
Each staff member should visually reflect state:

- idle (slight movement)
- moving
- compressing (loop animation)
- busy (med prep, IV, airway)
- confused (small visual cue)
- dismissed (fade/move out of room)

Simulate animations with:
- position changes
- scale pulsing
- rotation
- icon overlays if needed

-----------------------------------
CPR VISUALIZATION
-----------------------------------
When CPR is active:
- compressor visibly animates (vertical motion or pulsing)
- patient area shows rhythmic feedback
- CPR stops → animation stops immediately

-----------------------------------
SPEECH BUBBLES
-----------------------------------
Display speech bubbles above staff:

Examples:
- "Who’s giving epi?"
- "Charging!"
- "No pulse!"
- "I didn’t hear the dose."
- "Switching compressors."

Speech bubbles should:
- appear above correct staff
- fade after a few seconds
- not clutter screen excessively

-----------------------------------
CHAOS VISUALIZATION
-----------------------------------
Reflect chaos visually:

- overcrowding → more staff cluster near bed
- confusion → multiple speech bubbles
- failed orders → warning indicators
- high chaos → subtle screen tint or pulse effect

-----------------------------------
HUD OVERLAY
-----------------------------------
Keep a clean overlay showing:

- elapsed time
- rhythm
- pulse status
- ETCO2
- CPR status
- chaos meter
- compression fraction

Add a CRITICAL ALERT BANNER:

Always display the highest priority issue:
- "NO CPR IN PROGRESS"
- "SHOCKABLE RHYTHM"
- "NO IV ACCESS"
- "MEDICATION NOT OWNED"

-----------------------------------
ORDER VISIBILITY
-----------------------------------
Show active orders clearly:

- who owns the task
- current status (issued / heard / acknowledged / in_progress / completed)
- overdue indicators

Orders should not apply clinical effects until completed.

-----------------------------------
TEAM BEHAVIOR
-----------------------------------
Preserve and visualize behavior:

Staff should sometimes:
- not hear orders
- pick up wrong tasks
- delay execution
- duplicate actions
- ask for clarification

Make this visible via:
- movement hesitation
- speech bubbles
- delayed animations

-----------------------------------
PERFORMANCE RULES
-----------------------------------
- Do not use heavy 3D libraries
- Keep rendering lightweight
- Use React + CSS transforms or Canvas/SVG if needed
- Maintain smooth performance

-----------------------------------
DELIVERABLE
-----------------------------------
Produce a working 2.5D simulation where:

- I see a room with depth
- staff are positioned around the bed
- I can click staff to assign roles
- staff move when roles change
- CPR, shock, meds, airway are visible
- speech bubbles reflect communication
- chaos is visually noticeable
- simulation engine continues to run in real time

If needed, use simple shapes and icons instead of detailed graphics.
Focus on interaction and readability over visual realism.
